02. Project Vision & Objectives

Understanding the problem we're solving and the goals that guide development decisions.

The Problem

Plant diseases cause an estimated 10-16% loss in global crop production annually, with higher losses in developing regions where access to agricultural expertise is limited. For banana and wheat—two critical food crops—timely disease identification is essential but often delayed due to:

  • Lack of Expert Access: Agricultural extension officers are scarce in rural areas
  • Delayed Diagnosis: Lab testing takes days, allowing disease to spread
  • Misidentification: Similar symptoms across diseases lead to wrong treatments
  • Language Barriers: Technical resources often unavailable in local languages
  • Technology Gaps: Existing solutions require expensive equipment or connectivity

Our Solution

CropGuard democratizes access to AI-powered disease detection by:

Instant AI Diagnosis
Upload a leaf photo and get results in seconds—no waiting for lab results
Actionable Treatment Plans
Both chemical and organic options with specific product recommendations
Voice-First Design
Listen to results in English, Hindi, or Telugu for field use
Works on Any Phone
Responsive web app requires only a browser and camera

Project Objectives

# Objective Success Metric
1 Achieve high accuracy disease classification ≥90% accuracy on test dataset
2 Provide instant results <3 second prediction time
3 Support multiple languages English, Hindi, Telugu voice output
4 Mobile-first accessibility 100% responsive, <3MB initial load
5 Comprehensive treatment guidance Chemical + organic options for each disease
6 Admin analytics Dashboard with prediction trends
7 Easy deployment Single-command start with Docker support

Design Principles

  1. Farmer-Centric: Every feature decision considers the end-user—farmers with limited tech experience
  2. Offline-Capable: Core functionality works with intermittent connectivity
  3. Inclusive: Voice output and simple UI ensure accessibility for all literacy levels
  4. Transparent: Confidence scores help users understand prediction reliability
  5. Extensible: Modular architecture allows adding new crops and diseases

require("dotenv").config();
const express = require("express");
const cors = require("cors");
const path = require("path");
const connectDB = require("./config/db");

const authRoutes = require("./routes/authRoutes");
const userRoutes = require("./routes/userRoutes");
const predictionRoutes = require("./routes/predictionRoutes");
const chatRoutes = require("./routes/chatRoutes");
const dashboardRoutes = require("./routes/dashboardRoutes");
const translateRoutes = require("./routes/translateRoutes");

const app = express();
const PORT = process.env.PORT || 5000;

// Connect to MongoDB
connectDB();

// Middleware
app.use(cors());
app.use(express.json({ limit: "10mb" }));
app.use(express.urlencoded({ extended: true }));
app.use("/uploads", express.static(path.join(__dirname, "uploads")));

// Routes
app.use("/api", authRoutes);
app.use("/api", userRoutes);
app.use("/api", predictionRoutes);
app.use("/api", chatRoutes);
app.use("/api", dashboardRoutes);
app.use("/api", translateRoutes);

// Health check
app.get("/api/health", (_req, res) => {
  res.json({ status: "healthy", timestamp: new Date().toISOString() });
});

// Error handler
app.use((err, _req, res, _next) => {
  console.error("Server error:", err.message);
  res.status(500).json({ success: false, message: "Internal server error." });
});

app.listen(PORT, () => {
  console.log(`Server running on http://localhost:${PORT}`);
});

File Upload Configuration

backend/middleware/upload.js
const multer = require("multer");
const path = require("path");

const storage = multer.diskStorage({
  destination: (_req, _file, cb) => {
    cb(null, "uploads/");
  },
  filename: (_req, file, cb) => {
    const uniqueSuffix = Date.now() + "-" + Math.round(Math.random() * 1e9);
    cb(null, uniqueSuffix + path.extname(file.originalname));
  },
});

const fileFilter = (_req, file, cb) => {
  const allowedTypes = /jpeg|jpg|png|gif|webp/;
  const extname = allowedTypes.test(
    path.extname(file.originalname).toLowerCase()
  );
  const mimetype = allowedTypes.test(file.mimetype);

  if (extname && mimetype) {
    return cb(null, true);
  }
  cb(new Error("Only image files are allowed!"));
};

const upload = multer({
  storage,
  limits: { fileSize: 10 * 1024 * 1024 }, // 10MB limit
  fileFilter,
});

module.exports = upload;
File Storage: Uploaded images are stored in the backend/uploads/ directory with unique timestamps to prevent filename collisions.

Part VI: Disease Database

31. Banana Diseases Overview

Detailed information on all banana diseases detected by CropGuard.

🍌 Black Sigatoka (Mycosphaerella fijiensis)

Severity: HIGH

Black Sigatoka is a devastating leaf spot disease and one of the most economically important diseases of banana worldwide. It reduces photosynthesis and causes premature ripening.

Causes:

  • Fungus Mycosphaerella fijiensis
  • Wind-dispersed spores
  • Warm, humid conditions (24-28°C)
  • Poor air circulation

Symptoms:

  • Dark brown streaks on lower leaf surface
  • Large dark brown to black lesions
  • Premature leaf death
  • Uneven fruit ripening

🍌 Panama Disease / Fusarium Wilt (Fusarium oxysporum)

Severity: CRITICAL

Panama disease is a soil-borne fungal infection that attacks the vascular system, blocking water and nutrient flow. Tropical Race 4 (TR4) is the most dangerous strain.

Causes:

  • Soil-borne fungus Fusarium oxysporum f. sp. cubense
  • Contaminated soil, water, planting material
  • Persists in soil for decades

Symptoms:

  • Yellowing of older leaves from edges
  • Pseudostem splitting near base
  • Brown vascular discoloration
  • Wilting and collapse (skirt effect)

🍌 Bacterial Wilt / BXW (Xanthomonas campestris)

Severity: CRITICAL

Banana Bacterial Wilt can cause 100% crop loss if unmanaged. Spread by contaminated tools and insects visiting male buds.

Symptoms:

  • Progressive leaf wilting and yellowing
  • Brown, slimy bacterial ooze from cut stem
  • Premature fruit ripening
  • Complete plant collapse in 2-3 weeks

34. Installation Guide

Step-by-step instructions for installing CropGuard on a new system.

Prerequisites

Requirement Version Purpose
Node.js 18+ LTS Backend + Frontend runtime
Python 3.9-3.11 AI service runtime
MongoDB 6.x or Atlas Database
Git Any Clone repository

Step 1: Clone Repository

git clone https://github.com/Arif-Azeem786/Crop_Disease_Prediction.git
cd Crop_Disease_Prediction

Step 2: Setup AI Service

cd ai-model
python -m venv venv

# Windows
venv\Scripts\activate

# Linux/Mac
source venv/bin/activate

pip install -r requirements.txt
uvicorn main:app --reload --port 8000
Note: The trained model plant_disease_model.h5 is included. Leave this terminal running.

Step 3: Setup Backend

Open a new terminal:

cd backend
npm install

Create backend/.env:

PORT=5000
MONGODB_URI=mongodb://localhost:27017/plant_disease_db
AI_SERVICE_URL=http://127.0.0.1:8000
OPENAI_API_KEY=your_openai_api_key_here
npm start

Step 4: Setup Frontend

Open a new terminal:

cd frontend
npm install

Create frontend/.env:

REACT_APP_API_URL=http://localhost:5000/api
npm start

Step 5: Verify Installation

Service URL Expected Response
AI Service http://localhost:8000/health {"status": "healthy", "model_loaded": true}
Backend http://localhost:5000/api/health {"status": "healthy"}
Frontend http://localhost:3000 CropGuard homepage
